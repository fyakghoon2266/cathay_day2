## 🎯 工作流程

當使用者說「找客戶」「練習銷售」時：

1. 呼叫 `platform.find_customer`（api_key 用下方「我的 API key」段落的值）
2. 呼叫 `platform.start_session`，提供：
   - `customer_id`：上一步拿到的
   - `salesperson_persona`：見下方「我的人設」
   - `product_context`：見下方「我要賣的產品」
   - `api_key`：見下方「我的 API key」
   - `agent_name`：見下方「我的 Agent 名稱」
   - `team_display_name`：見下方「我們的隊名」
3. 看到客戶開場白後，用 `platform.send_message` 持續對話
4. 自主完成整場對話，**不要每輪都問使用者意見**
5. 對話結束後呼叫 `platform.end_session` 取得評分

### 💰 成交金額 = 客戶說的金額 × 你的表現倍率

每場對話結束會由 AI 教練打分（0-100）。客戶在對話中會說出他願意投入多少錢，
**最終成交金額 = 客戶說的金額 × 表現倍率**：

| 分數 | 倍率 | 意思 |
|---|---|---|
| 90-100 | 1.5-2.0x | 表現太好，客戶**加碼**投入更多 |
| 70-89 | 1.0-1.3x | 滿意，照說的金額或略多 |
| 50-69 | 0.6-0.9x | 猶豫，只投一部分 |
| < 50 | 0 | 客戶走人，不成交 |

例：客戶說「投 688 萬」，你拿 92 分 → 688萬 × 1.6 = **1100 萬**。
（金額不會超過客戶的總預算上限）

所以：攻略高手客戶（願意投入大）+ 拿高分（倍率高）= 排行榜爆衝。
結束時系統會明確告訴你「為什麼最終金額是這個數字」。

### 📊 評分的 4 個維度

1. **商品方向（40 分）**：推對方向是基本盤，推錯直接歸零
2. **興趣連結（20 分）**：接住客戶的興趣話題、有溫度
3. **個人觀點（20 分）**：客戶會問「你自己怎麼做」，要答得具體、一致
4. **性格應對（20 分）**：別踩客戶的性格地雷

→ 維度 2、3、4 就是 **skill 能大幅加分的地方**。

## 🛠 Onboarding 流程指令

**當使用者說「幫我設定 agent」「我第一次使用」「onboarding」「填 AGENTS.md」**
**或類似意圖時，請按以下流程執行：**

### 步驟 1：先確認隊名（不可跳過）

API key 已預設為共用的 `arena-2025`，不用問使用者。

但**隊名一定要問**，**而且要強調同組必須寫一模一樣的字**：

> 「請問你們組商量好的隊名是什麼？例如『閃電隊』『無敵海陸』。
>  ⚠️ 同組成員必須填**一模一樣**的字（包含空格、有沒有『隊』字），
>  不然系統會當成不同隊伍分開計分。請跟組員確認過再告訴我。」

如果使用者還沒跟組員確認，請停下來告訴他：
> 「請先跟組員確認好統一的隊名再回來，這個錯了就無法合併分數。」

如果使用者完全沒組員（自己一人）→ 也可以，他就一個人一隊。

### 步驟 2：問理專資訊（一題一題問，不要一次列完）

使用者答完前一題才問下一題。每題等使用者回應後再進下一題。

3. **Agent 名稱**：「你想取什麼名字當 agent？這會顯示在競技場的狗狗頭上。」
   - 範例提示：「金融達人 Amy」「理財柴犬」「收割者 Jason」
4. **人設**：「你想當什麼樣的理專？告訴我你的：年齡、經驗年數、專長、個性。」
   - 如果使用者答得太簡略（例如只說「我是理專」），追問細節
5. **產品**：「你擅長賣什麼類型的產品？保守型？穩健型？積極型？或是教育金、退休金規劃？」
6. **銷售風格**：「你的銷售風格偏向哪一種？溫和有耐心、專業犀利、親切幽默、誠實直接？」
7. **處理反對**：「客戶問你『你自己有買嗎？』『費用是不是太貴？』『跟其他銀行有什麼不一樣？』時，你會怎麼回？」

### 步驟 3：寫回 AGENTS.md

收集完上面所有答案後：

1. **使用 Edit 工具**，修改當前資料夾的 `AGENTS.md` 檔案
2. 把答案填進對應欄位：
   - `## 🔑 我的 API key` → 已預設 `arena-2025`，不用改
   - `## 🏆 我們的隊名` → 填入步驟 1 的隊名（一字不改地原樣寫入）
   - `## 🐶 我的 Agent 名稱` → 填入步驟 2 第 3 題
   - `## 🧑‍💼 我的人設` → 把步驟 2 第 4 題答案整理成 2-3 句話
   - `## 💼 我要賣的產品` → 把步驟 2 第 5 題答案整理成 2-3 句話
   - `## 🎨 銷售策略` → 結合步驟 2 第 6、7 題答案，整理成 4-5 條原則
3. **把每個欄位裡的「（請填寫——沒填寫會很難成交）」「（範例：...）」這種提示文字刪掉**，
   因為這些是給人看的，AI 看到反而會以為是內容
4. 保留欄位之外的所有結構（標題、提示說明等）

### 步驟 4：完成確認

修改完成後告訴使用者：

> ✅ AGENTS.md 已經填好了！
>
> 設定摘要：
> - 隊名：[隊名]
> - Agent：[agent_name]
> - 人設：[一句話人設]
>
> 接下來你可以對我說「**去找客戶練習**」開始競賽。
>
> ⚠️ 提示：你目前還沒建立任何 skill。第一次先用沒有 skill 的方式練練看，
> 失敗後看評分回饋，再決定要建什麼 skill。

### 注意事項

- **不要自作主張幫使用者寫人設**——一定要問他、用他的回答
- 如果使用者答得不完整、不知道怎麼答，可以給範例提示但不要直接代他寫
- 保持友善鼓勵的語氣，這是練習平台不是面試

### 建立流程

1. **讀模板**：使用 Read 工具讀取 `templates/example-interest-skill.md` 的內容當作參考模板
   （這個模板放在 `templates/` 資料夾，不會被 Codex 自動載入為 skill，所以不會誤觸發）
2. **建新資料夾**：根據對照表決定資料夾名（例如 `skills/trump-deals/`）
3. **訪問使用者**：一題一題問模板的 5 個段落（不要一次問完）：
   - 「你對 [主題] 的個人立場？」
   - 「你能聊的具體素材有哪些？至少 3 個」
   - 「你絕對不要假裝懂的事？」
   - 「客戶用閒聊問句開場時你想怎麼回？」
   - 「從興趣轉回理財話題的橋接句？」
4. **生成 SKILL.md**：使用 Edit/Write 工具建立檔案：
   - 替換所有 `REPLACE_客戶名` → 真實客戶名
   - 替換所有 `REPLACE_主題` → 真實主題
   - 替換所有 `REPLACE_關鍵字1/2/3` → 對應的觸發關鍵字（從上面對照表 + 你問使用者得到的）
   - 把每個段落的 placeholder 換成使用者的回答
5. **完成確認**：

   > ✅ 已建立 `skills/[資料夾名]/SKILL.md`
   >
   > 內容摘要：
   > - 觸發關鍵字：[列關鍵字]
   > - 個人立場：[一句話]
   > - 主要素材：[1 句]
   >
   > 請重啟 codex（/quit 後再進來）讓 skill 生效。
   > 然後再去找 [客戶名] 練習，看看效果！

### 注意事項

- **不要自己掰素材**——一定要問使用者真實的故事跟立場
- 如果使用者真的不知道某個話題（例如沒打過高爾夫），引導他寫「我不打但我客戶都打」這種誠實版本
